## Project Overview

This is the **More Green** Shopify theme — a Dawn v15.4.1 fork customized for a health supplement brand selling green powder SKUs (Sunflower Microgreens, Blueberry, Moringa, Wheatgrass). The working theme lives in the `theme/` directory and is deployed by uploading a POSIX-compliant zip to Shopify Admin.

## Deploying to Shopify

**Always use `tar` to zip — never `Compress-Archive`.** Windows `Compress-Archive` produces backslash paths that fail Shopify's upload validation ("missing template layout/theme.liquid").

```powershell
# From D:\More Green AGI\More Green Claude\
tar -a -c -f moregreen-theme.zip -C theme .
```

Then upload the zip in Shopify Admin → Online Store → Themes → Add theme → Upload zip file.

## Theme Structure

```
theme/
  assets/        # All CSS files (component-*.css, section-*.css) and JS (*.js) and SVG icons
  blocks/        # Reusable AI-generated blocks (ai_gen_block_*.liquid)
  config/        # settings_schema.json (theme settings definition) + settings_data.json (live values, auto-generated)
  layout/        # theme.liquid (root HTML shell), password.liquid
  locales/       # Translation strings; en.default.json is the source of truth
  sections/      # Page section templates (main-product.liquid, header.liquid, footer.liquid, etc.)
  snippets/      # Reusable partials (card-product.liquid, price.liquid, etc.)
  templates/     # Route-to-section mappings as JSON (index.json, product.json, etc.)
```

## AI-Generated Blocks (`theme/blocks/`)

Blocks named `ai_gen_block_*.liquid` follow a strict pattern:
- A `{% doc %}` comment at the top records the original generation prompt — preserve it.
- Each block uses `{% assign ai_gen_id = block.id | replace: '_', '' | downcase %}` to namespace all CSS classes uniquely (prevents collisions when the same block type appears multiple times on a page).
- All styles are scoped with the `ai_gen_id` suffix. When editing, always maintain this suffix pattern.
- JavaScript is wrapped in an IIFE and defines a `customElements.define` using the same id.

## Key Conventions

- **CSS scope**: Component-level styles live in `assets/component-*.css`; section-level styles in `assets/section-*.css`. Inline `{% style %}` blocks are used for dynamic (Liquid-interpolated) values only.
- **JS architecture**: Custom elements (Web Components) are used throughout. Each interactive UI piece extends `HTMLElement` and is defined via `customElements.define`. Inter-component communication uses the pub/sub system in `assets/pubsub.js`.
- **Cart API**: Add-to-cart calls go to `/cart/add.js` with JSON body `{ items: [{ id: variantId, quantity }] }`.
- **Fonts**: Header font is Archivo 600, body font is Inter 400 (loaded via Shopify CDN).
- **Brand colors**: Primary green `#08a045`, hover green `#067a35`. Page max-width is 1400px.
- **settings_data.json** is auto-generated by the Shopify theme editor — avoid manual edits; they will be overwritten.

## Influencer Discovery

**Always use `chrome-find-influencers` — never `find-influencers`.**
The Meta API hashtag search is blocked for this account. The Brave-based scraper is the only working discovery method.

```powershell
cd D:\More Green AGI\automation
python main.py chrome-find-influencers                        # all 15 default hashtags
python main.py chrome-find-influencers --hashtag <tag>        # specific hashtag(s)
python main.py chrome-find-influencers --limit 10             # cap profiles per hashtag
python main.py --dry-run chrome-find-influencers              # preview without writing to DB
```

After every discovery run, always sync to the Sheet:
```powershell
python main.py update-tracker
```

**Where results land:**
- **DB (source of truth):** `automation/db/pipeline.db` → `influencers` table, status = `discovered`
- **Sheet (Bharath's view):** https://docs.google.com/spreadsheets/d/12oJEJt3VR-Q2hlBRW49rQ2vDWSkOmxWz_i2c2-S3G0k

**Session file:** `automation/ig_session.json` — auto-saved after first login, gitignored. Delete it to force a fresh login.

## graphify

This project has a knowledge graph at graphify-out/ with god nodes, community structure, and cross-file relationships.

Rules:
- For codebase questions, first run `graphify query "<question>"` when graphify-out/graph.json exists. Use `graphify path "<A>" "<B>"` for relationships and `graphify explain "<concept>"` for focused concepts. These return a scoped subgraph, usually much smaller than GRAPH_REPORT.md or raw grep output.
- If graphify-out/wiki/index.md exists, use it for broad navigation instead of raw source browsing.
- Read graphify-out/GRAPH_REPORT.md only for broad architecture review or when query/path/explain do not surface enough context.
- After modifying code, run `graphify update .` to keep the graph current (AST-only, no API cost).
